## Architecture Notes

| Item | Detail |
|---|---|
| Background layers | `body::before` (ambient) and `body::after` (grain) are both `position: fixed` and `z-index: 0`, so they sit behind `#root` (`z-index: 1`) regardless of scroll or overflow on child elements. |
| `will-change` scope | Still only on the animating ambient layer (`body::before` gets `will-change: background-position` via the `ambientShift` animation). No change to Chat or other screens. |
| Sidebar glass performance | Sidebar now has `backdrop-filter`. This is the only `backdrop-filter` in the app shell — Chat's floating panel is the second. Total: 2 stacked layers maximum, within the performance budget. Main content wrapper uses solid `var(--bg-surface)` — no blur. |
| Sidebar `collapsed` state | Now starts collapsed (`true`) so the default view shows icon rail. `App.tsx` mirrors it via `onCollapsedChange` to drive `marginLeft` without prop drilling the sidebar's internal state. |
| No left-border indicator | Floating sidebar has no window edge to anchor a left-border indicator. Active item uses background highlight only, consistent with the spec. |
| Dashboard topbar | Topbar component in `App.tsx` continues to render the screen title and theme toggle — Dashboard screen itself no longer duplicates a topbar. |
