# GwenLand — `gwen chat --gui` Flag

**Date:** 2026-06-06  
**Scope:** `gwen-cli/packages/tui`

---

## Summary

Adds a `--gui` flag to `gwen chat` so users can launch the Tauri GUI window from the CLI
instead of the terminal chat TUI. Default behaviour is unchanged: `gwen chat` still opens
the existing ratatui interface. GUI support is behind an optional Cargo feature so the
default binary stays lean and does not link WebView2/Tauri unless explicitly requested.

---

## What Was Added

### `gwen chat --gui`

| Command | Behaviour |
|---|---|
| `gwen chat` | Existing TUI chat (no change) |
| `gwen chat --gui` | Launches GwenLand Tauri GUI window |
| `gwen chat --help` | Shows `--gui` in options list |

When `--gui` is passed and the binary was built **without** the `gui` feature, the CLI
exits with a clear message: rebuild with `--features gui`.

### Cargo feature `gui`

```toml
[features]
default = []
gui = ["dep:gwenland-gui"]

[dependencies]
gwenland-gui = { path = "../gui/src-tauri", optional = true }
```

- **Default build:** no Tauri/WebView2 linked; TUI-only binary.
- **GUI build:** `cargo build -p gwenland-tui --features gui` links `gwenland-gui` and
  enables `gwen_gui_lib::run()` (Tauri 2 + React frontend from Cycle 4).

---

## What Was Changed

| File | Change |
|---|---|
| `packages/tui/Cargo.toml` | Added `gui` feature and optional `gwenland-gui` dependency |
| `packages/tui/src/commands/chat.rs` | Added `gui: bool` to `ChatArgs`; early return in `run_chat_cmd()` when `--gui` |

---

## What Was NOT Changed

- Existing TUI chat logic (`run_chat_tui`, SSE streaming, pipe mode, rendering)
- Other CLI commands (`serve`, `run`, `doctor`, etc.)
- GUI package source (`packages/gui/`) — reused as-is via `gwen_gui_lib::run()`

---

## Architecture

```
gwen chat           → run_chat_cmd() → run_chat_tui()     → ratatui + SSE proxy
gwen chat --gui     → run_chat_cmd() → gwen_gui_lib::run() → Tauri 2 window
                                                              (React Chat UI)
```

Both paths share the same GwenLand GUI built in Cycle 4; the CLI does not duplicate
`tauri::generate_context!()` — that context lives in `gwenland-gui` where `tauri.conf.json`
and `build.rs` already exist.

---

## Build & Test

```bash
# Default (TUI only)
cargo build -p gwenland-tui
gwenland chat --help          # shows --gui flag
gwenland chat                 # TUI

# With GUI
cargo build -p gwenland-tui --features gui
gwenland chat --gui           # opens Tauri window
```

## Notes

- Windows requires the [WebView2 runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)
  for the GUI-enabled binary to start.
- `gwen start gui` stub in `main.rs` is unchanged; `--gui` on `gwen chat` is the supported
  entry point for this cycle.
