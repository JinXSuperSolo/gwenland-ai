# GwenCLI Change Log — 2026-06-01 12:00

## Overview

Cycle 3 CLI help text and `--version` output refresh. Every subcommand now has a short description (shown in `gwen --help`), a long description (shown in `gwen <cmd> --help`), and all flags documented with examples. `vergen` integration embeds the git short-SHA into the binary at build time. Five stub commands for unimplemented Cycle 3 features (`eval`, `convert`, `benchmark`, `config`, `update`) are registered so they appear in `gwen --help` with proper descriptions.

## Design Decisions

| Decision | Reason |
|---|---|
| `vergen` gitcl feature (not git2) | Does not require libgit2 native dependency. Works in any environment with `git` CLI in PATH (GitHub Actions, macOS, Linux, Windows). |
| `let _ = vergen::...emit()?` → kept `?` | CI always has git available (actions/checkout). Build error on gitless environment is acceptable and expected; use `let _ =` locally if needed. |
| Hidden legacy commands (`Start`, `Model`, `Setup`, `HubDataset`) | These are not in the Cycle 3 command surface. They continue to work when invoked directly but are excluded from `gwen --help` to keep the displayed command list clean. |
| Stub commands (`Eval`, `Convert`, `Benchmark`, `Config`, `Update`) | Registered in the `Commands` enum so they appear in `gwen --help` with descriptions. Dispatch arm prints "not yet implemented" and exits 0. |
| `option_env!("VERGEN_GIT_SHA").unwrap_or("unknown")` | Compile-time fallback for builds without a git SHA (e.g. source tarball releases). |
| `GlobalArgs` field reordered | Matches the spec's flag order: `--json`, `--non-interactive`, `--dry-run`, `--yes`. No functional change. |
