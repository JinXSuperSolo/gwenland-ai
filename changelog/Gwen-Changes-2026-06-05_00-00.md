## Critical Fixes

### C-1 — Placeholder pages for missing sidebar nav items

Train, Dataset, Eval, and Doctor nav items previously fell through silently to Dashboard
(the else branch). All four now navigate to dedicated placeholder pages with a clear
coming-soon message and the correct CLI alternative.

- **`src/pages/Train.tsx`** — New. FlaskConical icon. "Native Rust training via Candle ships in Cycle 5."
- **`src/pages/Dataset.tsx`** — New. Database icon. "Dataset management ships in Cycle 5."
- **`src/pages/Eval.tsx`** — New. BarChart2 icon. "Model evaluation ships in Cycle 6."
- **`src/pages/Doctor.tsx`** — New. Stethoscope icon. "Environment diagnostics ships in Cycle 4 CLI. GUI view coming soon."
- **`src/App.tsx`** — Wired all four routes in the switch block.
- **`src/components/Sidebar.tsx`** — Replaced the live green dot badge on Doctor with a static "Soon" text badge. The green dot implied live health data that does not exist.

### C-2 / C-3 — Config values wired into all hooks

Previously every hook hardcoded its target URL. Changing the proxy port in Settings had
zero effect. All hooks now read from a single `ConfigContext`.

- **`src/context/ConfigContext.tsx`** — New. Wraps `useSettings` once at the app root; all hooks read via `useConfig()`. Exposes `config`, `update`, `reset`, `loading`, `saved`.
- **`src/hooks/useProxyHealth.ts`** — Reads `config.proxyPort`; removed hardcoded `1136`.
- **`src/hooks/useChat.ts`** — Reads `config.proxyPort` and `config.streamTimeout`; adds `AbortSignal.any([abort.signal, AbortSignal.timeout(timeoutMs)])` for stream timeout.
- **`src/hooks/useModels.ts`** — Reads `config.ollamaHost`; removed hardcoded `localhost:11434`.
- **`src/hooks/usePullModel.ts`** — Reads `config.ollamaHost`; removed hardcoded `localhost:11434`.
- **`src/hooks/useDeleteModel.ts`** — Reads `config.ollamaHost`; added `onError` callback so failures surface as toasts instead of silent console.error.
- **`src/hooks/useSettings.ts`** — Added TODO comments for `launchAtStartup`, `startMinimized`, `startupMode`, `language` — these are saved to disk but require OS-level integration not yet implemented.
- **`src/pages/Settings.tsx`** — Switched from direct `useSettings()` to `useConfig()` for config reads/writes.
- **`src/App.tsx`** — Added `autoStartProxy` effect on mount: spawns `gwen serve` once if `config.autoStartProxy` is true.
- **`src/main.tsx`** — Wrapped App in `<ConfigProvider>`.
- **`src/pages/Chat.tsx`** — Reads `config.proxyPort` for the port label in the topbar SSE badge.

## High Fixes

### H-1 — Dead affordances removed

- **`src/pages/Chat.tsx`** — Removed Export (Download) and More Options (MoreHorizontal) icon buttons from the Chat topbar. Neither had an onClick; their presence implied functionality that did not exist.
- **`src/components/chat/ChatInput.tsx`** — Replaced the visible disabled paperclip button with `{null}`. A disabled button with `cursor: default` still signals "this does something" to users.
- **`src/components/RecentSessions.tsx`** — Removed `cursor: pointer` and hover background from session rows; added `opacity: 0.5` to signal non-interactivity. Session loading from JSONL is deferred to a later cycle. Removed unused `useState` import.
- **`src/pages/Dashboard.tsx`** — Passes `onNewChat={() => onNavigate?.('chat')}` to RecentSessions. The New Chat button now navigates to Chat.

### H-2 — Error boundary

- **`src/components/ErrorBoundary.tsx`** — New class component. Catches unhandled React errors, shows a recovery screen with the error message and a "Try again" button (resets error state without a full page reload).
- **`src/main.tsx`** — `<ErrorBoundary>` wraps the entire app, outside `<ConfigProvider>`.

### H-3 — User-visible errors (no more silent failures)

- **`src/hooks/useToast.ts`** — New. Hand-rolled; no external library. Three variants: error / success / warning. Auto-dismiss after 4s. `dismiss(id)` for manual close.
- **`src/components/ToastStack.tsx`** — New. Fixed bottom-right, z-index 100. AlertTriangle / CheckCircle icons per variant; dismiss button.
- **`src/App.tsx`** — Owns `useToast()`, renders `<ToastStack>` at root, passes `toast` fn as prop to pages that need it.
- **`src/components/settings/DangerZone.tsx`** — Added `onError` prop; calls it with a warning message when a session file delete fails instead of swallowing the error.
- **`src/components/settings/PathSetting.tsx`** — Added `onError` prop; calls it with `'Could not open folder'` instead of silently swallowing `openPath` failures.
- **`src/components/settings/AboutCard.tsx`** — Added `onError` prop; calls it with `'Could not open browser'` instead of silently swallowing `openUrl` failures.
- **`src/pages/Settings.tsx`** — Passes `toast` down to DangerZone, PathSetting, and AboutCard.
- **`src/pages/Models.tsx`** — Passes `onError` to `useDeleteModel`; error now surfaces as a toast.

### H-4 — Dashboard live data

- **`src/pages/Dashboard.tsx`** — Replaced all hardcoded strings. Active model reads from `useModels().activeModel` (shows `—` when Ollama is offline). Models Installed reads from `models.length`. Proxy Status reads from `useProxyHealth().alive`. Binary Size and Cold Start remain as static build facts (Criterion-verified / build artifact).

## Medium Fixes

### M-1 — Race conditions

- **`src/hooks/useModels.ts`** — Auto-select only fires when `prev` model is not already in the list; user's explicit selection is preserved across refetches.
- **`src/hooks/useProxyHealth.ts`** — Initial state is `alive: true` (optimistic). Avoids a false "proxy offline" flash for up to 3s on app open when the proxy is already running.

### M-2 — Accessibility

- **`src/components/RecentSessions.tsx`** — Added `aria-label` to session rows.
- **`src/components/chat/MessageList.tsx`** — Fixed hint chip mouse handlers to use `e.currentTarget` instead of `e.target as HTMLButtonElement`.
- **`src/components/models/RegistryTabs.tsx`** — Added `role="tablist"`, `role="tab"`, `aria-selected`, `aria-disabled` to tab buttons.
- **`src/components/models/ModelCard.tsx`** — Added `ariaLabel` prop to `IconAction`; Info button gets `"Model info"`, Delete button gets `"Delete {name}"`. Added `aria-disabled` to disabled state.
- **`src/components/chat/ModelPicker.tsx`** — Added `role="listbox"` wrapper, `role="option"` and `aria-selected` on model rows. Fixed pull button hover to use `e.currentTarget`.

### M-3 — Double resize call in ChatInput

- **`src/components/chat/ChatInput.tsx`** — Removed the `resize()` call from `onChange`. `onInput` is the correct handler for layout reflow; `onChange` only updates React state. Calling both fired two style mutations per keystroke.

## Architecture Notes

| Item | Detail |
|---|---|
| Config single source | One `useSettings()` call at the `ConfigProvider` level. All hooks call `useConfig()` — no file is read twice, no debounce timer races. |
| Optimistic proxy health | `alive` starts `true` so new Chat sessions don't see a false "proxy offline" flash. The real check runs within 3s and flips to `false` only on confirmed failure. |
| AbortSignal.any | `useChat` now applies both user abort and a wall-clock timeout in a single signal. A stalled stream auto-aborts after `config.streamTimeout` seconds without needing a separate timer. |
| Toast ownership | `useToast` lives in App, not in a context. Toast is passed as a prop to pages (1-hop max). If toast propagation grows beyond 2 hops, promote to context. |
| Dead affordances rule | If a button has no onClick, it must either be removed or rendered as `{null}`. A disabled visible button with `cursor: default` still implies functionality. |
| Session rows (Option A) | Session rows are `opacity: 0.5` with no cursor/hover. JSONL session loading is deferred; Option B (clickable) requires a session format definition that doesn't exist yet. |
