## Architecture Notes

| Item | Detail |
|---|---|
| Navigation | Single `useState<Screen>` in `App.tsx` — no react-router. Tauri's WebView has no address bar so URL routing adds complexity with no benefit. |
| Lazy loading | All four screens `React.lazy()` behind `<Suspense>`. Initial JS chunk is minimal. |
| No `<form>` tags | All submit actions use `onClick` / `onKeyDown` handlers per Tauri requirements. |
| SSE streaming | `EventSource` at `127.0.0.1:1136/stream` bypasses Tauri IPC — no serialisation overhead per token. DOM updates via single `setMessages` call per chunk. |
| Glass performance | `backdrop-filter` applied only to the Chat floating panel. Sidebar uses solid `var(--bg-surface)`. Max 2 backdrop-filter layers; blur capped at 16px. |
| `will-change` scope | Only on `.bg-ambient` animated gradient (`will-change: background-position`). Not set globally. |
| Mock data coherence | Stub sessions reference Mistral-7B, matching `get_stats()` active model and `get_config()` default model. |
| `gwen-core` dep | `default-features = false` excludes GPU/candle backends, keeping GUI binary lean. |
| `build.rs` | Required by Tauri 2's `tauri-build` — was absent from the spec but mandatory for `cargo build` to succeed. |
