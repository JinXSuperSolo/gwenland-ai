### Design Decisions

| Decision | Reason |
|---|---|
| `gwen-tui` binary name (not `gwen`) | Clap `name = "gwen"` is the display name; the actual binary on disk is `gwen-tui` per `Cargo.toml` package name. |
| `-q q4_k_m` hardcoded on fetch | `fetch.rs` returns `QuantRequiredNonInteractive` error when stdout is not a TTY and `-q` is absent. CI stdout is never a TTY. |
| `--dry-run` on train (not `--max-steps`) | `TrainConfig` and `TrainArgs` have no `--max-steps` flag. Dry-run is the only available CI guard — it invokes `run_train_with_opts(dry_run=true)` which skips the Python subprocess entirely. |
| `fail-fast: true` | Pipeline steps are sequential (each depends on the previous). Any failure invalidates downstream steps; continuing wastes runner minutes. |

### Test Cases

#### Test A — `test_serve_subprocess_teardown`

Spawns `gwen-tui serve tinyllama/TinyLlama-1.1B --port PORT --non-interactive`. Polls `try_wait()` every 200 ms for 30 s.

- **CI path (mistralrs absent):** binary exits within ~1 s with code 1 (`EXIT_ERROR`). Test asserts exit code ∈ `{0,1,2,3}`.
- **Live path (mistralrs present):** process survives 30 s → SIGTERM sent → `KillOnDrop` guard fires on drop.
- **Zombie assertion:** `KillOnDrop` (see below) guarantees kill even on panic. No `wait()` needed.

#### Test B — `test_proxy_sse_stream`

Tests the full SSE client contract using a real in-process HTTP server (raw `TcpListener`, no framework):

1. Start fake upstream on ephemeral port (answers `GET /health → 200`, `POST /api/chat → chunked SSE stream`).
2. Poll `/health` — max 30 s.
3. `POST /api/chat` via `reqwest-eventsource::EventSource`.
4. Assert ≥1 token received, content matches `["Hello", " world", "!"]`.
5. Assert stream closes cleanly (`done=true` or `StreamEnded`).
6. Abort upstream task — no zombie.

**Why not test through the Actix proxy:** `proxy::start()` binds to the private constant `"127.0.0.1:1136"` with no parameterised overload. Patching `PROXY_BIND_ADDR` would require a feature flag or private API change. The proxy is a transparent byte pipe — the SSE format it forwards is identical to what the upstream emits. This test validates the SSE client contract that the proxy would serve verbatim.

#### Test C — `test_serve_health_chat_sigterm`

The canonical spec-flow test (matches the `@TODO` items exactly):

| Step | Action |
|---|---|
| 1 Spawn | `gwen-tui serve tinyllama --port PORT --non-interactive` |
| 2 Poll health | `GET http://127.0.0.1:PORT/health` every 500 ms, max 30 s; bail immediately if `try_wait()` returns Some |
| 3 Stream chat | `POST /gwencli/chat` via `EventSource` (only if became_ready=true) |
| 4 Assert tokens | `tokens_received ≥ 1`, `stream_ok = true` |
| 5 SIGTERM | Unix: `kill -15 PID`; Windows: `taskkill /PID PID /F` |
| 6 No zombie | `try_wait()` loop — 5 s deadline → exit code ∈ `{0,1,2,3}` or None (signal kill) |

### `KillOnDrop` RAII Guard

```rust
struct KillOnDrop(tokio::process::Child);

impl Drop for KillOnDrop {
    fn drop(&mut self) {
        let _ = self.0.start_kill(); // non-blocking; OS reaps asynchronously
    }
}
```

- `start_kill()` is synchronous — no `.await` needed from `Drop`.
- Belt-and-suspenders: `Command::kill_on_drop(true)` also sets the OS kill-on-parent-exit flag.
- Both mechanisms together guarantee no zombie even on test panic.

### Fake Upstream SSE Format

Matches Ollama wire format (the format `proxy.rs` and `chat.rs` both consume):

```
HTTP/1.1 200 OK
Content-Type: text/event-stream
Transfer-Encoding: chunked

data: {"message":{"role":"assistant","content":"Hello"},"done":false}

data: {"message":{"role":"assistant","content":" world"},"done":false}

data: {"message":{"role":"assistant","content":"!"},"done":false}

data: {"message":{"role":"assistant","content":""},"done":true}

0

```

### Port Isolation

```rust
fn free_port() -> u16 {
    std::net::TcpListener::bind("127.0.0.1:0")
        .expect("bind :0")
        .local_addr().expect("local_addr").port()
}
```

OS assigns an ephemeral port — no hardcoded port in any test. The `kill_on_drop(true)` flag on the subprocess combined with ephemeral port selection prevents CI port conflicts across parallel matrix jobs.

## Architecture Notes (shared)

| Item | Detail |
|---|---|
| `/health` endpoint owner | Served by `mistralrs-server` subprocess, **not** the Actix proxy. `proxy.rs` only routes `POST /gwencli/chat`. `gwen_core::platform::serve::wait_for_ready()` polls `/health` directly on mistralrs. |
| Proxy bind address | Hardcoded `"127.0.0.1:1136"` private constant in `proxy.rs`. Not parameterisable via the public API. |
| Binary name on disk | `gwen-tui` (Cargo package name). Clap display name is `gwen`. All CI invocations use `gwen-tui`. |
| `train` CLI — no `--max-steps` | `TrainArgs` and `TrainConfig` have no `max_steps` field. The 5-step cap is `--dry-run` (no subprocess). `ci_train_config.yaml` sets `epochs: 1` as a secondary guard. |
| `hub push` dry-run | `run_push()` in `hub_model.rs` does not inspect `mode.dry_run`. Global `--dry-run` is passed; safety relies on scoped repo name + `HF_TOKEN` being absent or read-only on CI. |
