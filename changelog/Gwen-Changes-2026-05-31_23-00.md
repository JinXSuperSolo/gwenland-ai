# GwenCLI Change Log — 2026-05-31 23:00

## Overview

Two infrastructure tickets implemented in this session: **INFRA-01** (JSON → TOML config migration with sectioned `GwenConfig`) and **INFRA-02** (cross-platform path centralisation via the `directories` crate, replacing all hardcoded `~/.config`, `~/.gwen`, `AppData`, and `HOME`/`USERPROFILE` env-var constructions with `GwenPaths::*`).

---

## INFRA-01 — JSON → TOML Config Migration

### New Files

| File | Description |
|---|---|
| `packages/core/src/storage/config.rs` | Full rewrite. `GwenConfig` now has three section structs: `GeneralConfig` (`last_used_model`, `default_port`), `AiConfig` (`compression`, `token_budget`, `strategy`), `AuthConfig` (empty, reserved — HF token stays in OS keyring only). `load()` auto-migrates `config.json → config.toml` on first run, prints `✦ Config migrated to TOML format.`, then deletes the old file. `save()` writes pretty TOML. `get(key)` / `set(key, value)` accept dotted keys (`"general.last_used_model"`, `"ai.token_budget"`, etc.). Free functions `read_last_used_model()`, `save_last_used_model()`, `get()`, `set()` preserved for callers. |

### Modified Files

| File | Change |
|---|---|
| `packages/core/Cargo.toml` | Added `toml = "0.8"`. |
| `packages/core/src/storage/paths.rs` | Added `config_toml_path()` and `models_toml_path()` shim helpers. |
| `packages/core/src/storage/registry.rs` | Migrated from `models.json` → `models.toml`. `load()` auto-migrates legacy JSON (both structured `RegistryFile` and bare `Vec` written by old `fetch.rs`). `save()` uses `toml::to_string_pretty`. `rebuild_from_disk()` updated to write `models.toml`. |
| `packages/tui/src/commands/setup.rs` | Replaced raw `serde_json` scaffold with `GwenConfig::default().save()` — writes `config.toml` on first run. |
| `packages/tui/src/commands/fetch.rs` | Removed local `GwenConfig` struct. `load_hf_token` / `save_hf_token` now use OS keyring exclusively (consistent with `hub_model`). Registry write uses `ModelRegistry::upsert()` → `models.toml`. |
| `packages/tui/Cargo.toml` | Added `keyring = "3"` (needed by `fetch.rs` after token refactor). |

### TOML Config Structure

```toml
[general]
last_used_model = ""
default_port = 1136

[ai]
compression = true
token_budget = 4096
strategy = "tfidf"

[auth]
# hf_token stored in OS keyring, not here
```

### Migration Flow

```
startup
  config.toml exists?
    yes → load normally
    no  → config.json exists?
            yes → parse, copy fields → write config.toml → delete config.json
                  print "✦ Config migrated to TOML format."
            no  → use GwenConfig::default()
```

### Token Security Model (unchanged)

HF token is **never written to any config file**. Resolution order:

1. OS keyring (`gwen-cli` / `hf_token` entry via `keyring` crate)
2. `HF_TOKEN` environment variable

### `get` / `set` Key Reference

| Key | Type | Default |
|---|---|---|
| `general.last_used_model` | `String` | `""` |
| `general.default_port` | `u16` | `1136` |
| `ai.compression` | `bool` | `true` |
| `ai.token_budget` | `u32` | `4096` |
| `ai.strategy` | `String` | `"tfidf"` |

`--json` flag on `gwen config` commands still outputs JSON. TOML is only for the config file.

## INFRA-02 — Cross-Platform Path Centralisation

### Modified Files

| File | Change |
|---|---|
| `packages/core/Cargo.toml` | Added `directories = "5"`. Removed implicit `dirs` usage from all storage/engine modules. |
| `packages/core/src/storage/paths.rs` | Full rewrite. `GwenPaths` struct with 9 static methods backed by `ProjectDirs::from("dev", "JinXSuper", "gwen")` from the `directories` crate. `GWEN_HOME` env var override preserved for CI and portable installs. Backwards-compat shims `gwen_config_dir()`, `config_toml_path()`, `models_toml_path()` kept for callers not yet migrated. |
| `packages/core/src/storage/config.rs` | `toml_path()` → `GwenPaths::config_file()`. `json_path()` → `GwenPaths::config_dir().join("config.json")`. `save()` creates dir via `GwenPaths::config_dir()`. |
| `packages/core/src/storage/registry.rs` | `load()` → `GwenPaths::models_file()` and `GwenPaths::config_dir().join("models.json")`. `rebuild_from_disk()` → `GwenPaths::models_file()` and `GwenPaths::models_dir()`. |
| `packages/core/src/storage/history.rs` | `ConversationHistory::new()` → `GwenPaths::history_file()`. Deleted legacy `gwen_dir()` (used raw `HOME`/`USERPROFILE` env vars). |
| `packages/core/src/storage/session.rs` | Both `session_dir()` call sites (in `SessionLog::create()` and `find_last_crashed_session()`) → `GwenPaths::session_dir()`. Deleted legacy `session_dir()` (used `dirs::home_dir()` + `.gwen/session`). |
| `packages/core/src/engine/tokenizer.rs` | `get_active_model_from_config()` now reads `GwenConfig::load().general.last_used_model` instead of parsing `~/.gwen/config/config.json` via `get_home_dir()`. Deleted `get_home_dir()` helper (used `USERPROFILE`/`HOME` env vars). |
| `packages/core/src/engine/windowing.rs` | `WindowConfig::load()` reads `[ai]` section from `GwenConfig::load()`. Deleted `load_from_file()` and `config_json_path()` (used `dirs::home_dir()` + `.gwen/config.json`). Removed `fs`/`PathBuf`/`dirs` imports. |
| `packages/tui/src/commands/fetch.rs` | `models_dir()` via `GwenPaths::models_dir()`. Temp files via `GwenPaths::tmp_dir()`. `load_hf_token` / `save_hf_token` no longer take `&Path`. Removed unused `Path` import. |

### `GwenPaths` API Reference

| Method | Linux | macOS | Windows |
|---|---|---|---|
| `GwenPaths::config_dir()` | `~/.config/gwen/` | `~/Library/Application Support/gwen/` | `AppData\Roaming\gwen\` |
| `GwenPaths::config_file()` | `~/.config/gwen/config.toml` | `…/gwen/config.toml` | `…\gwen\config.toml` |
| `GwenPaths::models_file()` | `~/.config/gwen/models.toml` | `…/gwen/models.toml` | `…\gwen\models.toml` |
| `GwenPaths::models_dir()` | `~/.config/gwen/models/` | `…/gwen/models/` | `…\gwen\models\` |
| `GwenPaths::cache_dir()` | `~/.cache/gwen/` | `~/Library/Caches/gwen/` | `AppData\Local\gwen\cache\` |
| `GwenPaths::tmp_dir()` | `~/.cache/gwen/tmp/` | `…/gwen/tmp/` | `…\gwen\cache\tmp\` |
| `GwenPaths::eval_results_dir()` | `~/.config/gwen/eval_results/` | `…/gwen/eval_results/` | `…\gwen\eval_results\` |
| `GwenPaths::history_file()` | `~/.config/gwen/history.jsonl` | `…/gwen/history.jsonl` | `…\gwen\history.jsonl` |
| `GwenPaths::session_dir()` | `~/.cache/gwen/sessions/` | `…/gwen/sessions/` | `…\gwen\cache\sessions\` |

`GWEN_HOME` environment variable overrides `config_dir()` (and derives `cache_dir()` as `$GWEN_HOME/cache`).
