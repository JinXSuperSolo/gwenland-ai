# GwenCLI Change Log — 2026-06-02 12:00

## Overview

Implements the full `gwen convert` command across six new files. All conversion
logic (GGUF binary parsing, two-mode dequantisation, SafeTensors writing) lives
in `gwen-core`; `gwen-tui` calls only typed functions and owns the progress
printing and summary rendering. Supports `gwen convert gguf <PATH>` (standard
linear dequant), `gwen convert gguf <PATH> --euler` (GwenTensor cosine
projection), and `gwen convert st <PATH>` (stub, prints "coming soon"). Zero
new errors, zero new warnings introduced.

## New / Modified Files

| File | Change |
|---|---|
| `packages/core/src/convert/mod.rs` | **New file.** Module registry for the three convert sub-modules and the public `convert_gguf()` entry-point function. Defines `ConvertResult` (tensors_converted, mode, output_path, elapsed_secs, euler_warning) and `TensorProgress` (per-tensor callback payload). Owns the output path derivation (same dir + stem + `.safetensors` extension) and the Euler sweet-spot percentage accumulation logic. |
| `packages/core/src/convert/gguf_parser.rs` | **New file.** Hand-written minimal GGUF v1/v2/v3 binary parser. Reads magic, version, tensor count, KV count; skips all KV metadata entries without interpreting them (recursive `skip_kv_value` covers all 13 GGUF value types including nested arrays); reads tensor info section (name, shape, ggml_type, data_offset); seeks to the data segment respecting 32-byte alignment; loads each tensor's raw bytes sequentially. Defines `GgufDtype` (F32, F16, Q8_0, Q4_0), `TensorInfo`, `GgufFile`. Includes a manual `read_f16_as_f32` bit-manipulation function (exponent rebias + mantissa shift) to avoid depending on the `half` crate. `raw_size_bytes` computes exact on-disk byte counts per dtype including Q-block headers. |
| `packages/core/src/convert/dequant.rs` | **New file.** Two dequantisation modes. `DequantMode` enum (`Standard`, `Euler`). Standard Q8_0: 34-byte blocks, W = i8×scale (symmetric quantisation, zero_point = 0 per GGML convention). Standard Q4_0: 18-byte blocks, nibble unpacking with -8 offset, W = nibble×scale. Euler mode: `euler_dequant_block` — θ_i = (X×π)/max_bound, W = cos(θ)×δ_b/φ where φ = 1.6180339 (Golden Ratio). Full derivation comment explains why cosine projection was chosen over sigmoid/tanh, what each variable represents, and the rationale for per-block max_bound vs dtype-theoretical bound. Degenerate case (max_bound == 0, fully-pruned block) outputs 0.0 for all elements. F32/F16 pass-through modes for unquantised tensors. |
| `packages/core/src/convert/writer.rs` | **New file.** Manual SafeTensors output writer. Avoids the optional `safetensors` crate (which requires the `candle` feature) so the convert command works in the default feature set. Builds the JSON header manually (avoids a serde path through this module): `__metadata__:{}` plus per-tensor entries with `dtype:"F32"`, `shape:[...]`, `data_offsets:[start,end]`. Writes 8-byte LE header size prefix, JSON header bytes, then raw LE f32 data for each tensor in order via a `BufWriter`. |
| `packages/tui/src/commands/convert.rs` | **New file.** `ConvertArgs` (clap `Args`), `ConvertCommands` enum (`Gguf(GgufArgs)`, `St(StArgs)`), `GgufArgs` (`path: PathBuf`, `--euler: bool`), `StArgs` (`path: PathBuf`). `run_convert_cmd` dispatches to `run_gguf` or `run_st`. `run_gguf` calls `convert_gguf` with an inline closure that calls `print_tensor_line` per tensor. Progress format: `Converting [3/42] <ORANGE>name<RESET> (Q4_0, 4096×32000) ... ✓ 124ms`. Gwen Orange = `\x1b[38;2;255;140;66m`. Summary box uses `✦ Conversion Complete` header. Euler warning printed before summary box when > 20% of weights fall outside [-0.309, 0.309]. `run_st` prints `"coming soon"`. |
| `packages/core/src/lib.rs` | **Modified:** Added `pub mod convert;` before `pub mod dataset`. |
| `packages/tui/src/commands/mod.rs` | **Modified:** Added `pub mod convert;` (alphabetical order). |
| `packages/tui/src/main.rs` | **Modified:** Replaced `Convert` unit variant with `Convert(commands::convert::ConvertArgs)`. Removed `Convert` from the `Commands::Convert \| Commands::Benchmark` stub match arm (Benchmark remains). Added `Commands::Convert(args) => { commands::convert::run_convert_cmd(args); std::process::exit(0); }`. |

## Architecture Notes

| Item | Detail |
|---|---|
| Crate boundary | All format parsing, dequantisation math, and file I/O live in `gwen-core`. `gwen-tui`'s `convert.rs` calls only `gwen_core::convert::{convert_gguf, DequantMode, TensorProgress}` — it never touches `std::io`, nibble unpacking, or the f16 bit-manipulation directly. |
| No new crate dependencies | The GGUF parser is hand-written (standard-library I/O only). The SafeTensors writer is hand-written (no `safetensors` crate). The `half` crate is not used — f16→f32 conversion is done with a 10-line bit-manipulation function documented with a derivation comment. This keeps the default feature binary size unchanged. |
| Manual GGUF parser vs crate | A `gguf` crate exists but is unstable and would pull in additional transitive dependencies. We need only a subset of the spec (magic/version check, skip KV metadata, read tensor infos, seek to data). The manual parser is ~300 lines including all comments and covers GGUF v1–v3. |
| Euler sweet-spot warning | Accumulates `euler_outside_sweet` and `euler_total_weights` across all tensors in `mod.rs`, then computes the percentage at the end. Checking per-block would give the same result but would require an extra accumulator in `dequant.rs`. Keeping the accumulation in `mod.rs` keeps the dequant functions pure (no side effects beyond returning `Vec<f32>`). |
| SafeTensors JSON header | Built as a `String` with manual concatenation. Using `serde_json` would work but would introduce a serde path in `writer.rs` that is otherwise unnecessary. The header is small (O(tensor_count) bytes, not O(data_size)) so string allocation is negligible. |
| `gen` keyword | Not encountered in this session's code (no variables named `gen`). No edition-2024 keyword collisions introduced. |
