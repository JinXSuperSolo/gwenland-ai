## Architecture Notes

| Item | Detail |
|---|---|
| Font self-hosting | `geist/font/sans` and `/mono` exports use `next/font` and crash outside Next.js. Variable woff2 files extracted from the package tarball and served from `public/fonts/` with `@font-face` ŌĆö consistent with the scaffold memory note. |
| Frameless window | `decorations: false` removes the OS titlebar. `data-tauri-drag-region` on the topbar `<header>` is the sole drag handle. Window controls are custom React buttons calling `@tauri-apps/api/window`. |
| Capabilities | Tauri v2 denies all by default. `capabilities/default.json` grants exactly the four permissions the window-control buttons need ŌĆö nothing more. |
| `[lib]` section | Added to `Cargo.toml` even though the spec omitted it. Required for `main.rs`'s `gwen_gui_lib::run()` crate name to resolve and for mobile build compatibility. |
| Release profile | The `[profile.release]` block in `src-tauri/Cargo.toml` is silently ignored by Cargo ŌĆö workspace-level profiles in `gwen-cli/Cargo.toml` take precedence. Effective release settings are `opt-level = "z"`, `lto = "fat"`, `panic = "abort"` from the root. |
| Icon pruning | `tauri icon` generates ~54 files (Android, iOS, Windows Store sets). All pruned immediately ŌĆö only `32x32.png`, `icon.ico`, `icon.png` retained for the Windows desktop target. |
| Workspace membership | `gwen-cli/Cargo.toml` already listed `packages/gui/src-tauri` as a workspace member before this session. No workspace config changes needed. |
