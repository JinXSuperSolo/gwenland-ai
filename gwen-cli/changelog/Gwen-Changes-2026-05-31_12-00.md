### `vram.rs` вЂ” What Was Implemented

#### `VramEntry` + `VRAM_TABLE`

Static lookup table with 18 entries covering 6 model size tiers Г— 3 quantization levels:

| Tier | q4 base VRAM | q8 base VRAM | f16 base VRAM |
|---|---|---|---|
| 1B  (e.g. SmolLM 1.7B)              | 0.9 GB  | 1.7 GB  | 3.4 GB   |
| 3B  (e.g. Phi-3 Mini, Qwen 3B)      | 2.0 GB  | 3.5 GB  | 6.0 GB   |
| 7B  (e.g. Mistral 7B, LLaMA 3.1 8B) | 4.2 GB  | 7.8 GB  | 14.0 GB  |
| 13B (e.g. LLaMA 2 13B, Qwen 14B)    | 7.8 GB  | 14.0 GB | 26.0 GB  |
| 27B (e.g. Gemma 2 27B, Qwen 32B)    | 15.0 GB | 27.0 GB | 54.0 GB  |
| 70B (e.g. LLaMA 3.1 70B, Qwen 72B)  | 38.0 GB | 70.0 GB | 140.0 GB |

All labels include human-readable examples in parentheses.

#### `VramEstimate` struct

| Field | Type | Meaning |
|---|---|---|
| `model_label` | `String` | Human-readable label from `VRAM_TABLE` |
| `base_gb` | `f32` | Base model VRAM from table |
| `lora_gb` | `f32` | `max(lora_r Г— lora_alpha Г— 0.001, 0.1)` |
| `activation_gb` | `f32` | `max(batch_size Г— max_seq_len Г— 0.001, 0.5)` |
| `optimizer_gb` | `f32` | `base Г— 0.4` for `adamw_8bit`, `base Г— 0.8` full AdamW |
| `safety_gb` | `f32` | 20% of subtotal |
| `total_gb` | `f32` | `subtotal Г— 1.2` |
| `available_gb` | `Option<f32>` | From GPU detection; `None` if no GPU found |
| `gpu_name` | `Option<String>` | GPU model name from `nvidia-smi` |
| `fits` | `bool` | `available_gb >= total_gb` |

#### VRAM Component Formula

```
lora_gb        = max(lora_r Г— lora_alpha Г— 0.001, 0.1)
activation_gb  = max(batch_size Г— max_seq_len Г— 0.001, 0.5)
optimizer_gb   = base_gb Г— 0.4   (adamw_8bit)
               = base_gb Г— 0.8   (full AdamW)
subtotal       = base_gb + lora_gb + activation_gb + optimizer_gb
safety_gb      = subtotal Г— 0.2
total_gb       = subtotal + safety_gb   (= subtotal Г— 1.2)
```

#### Model Matching

`parse_param_billions(model_name)` scans the model ID string for numeric patterns followed by `b`/`B` (e.g. `"7b"`, `"72B"`, `"13B"`). Falls back to `7.0` if no match found.

`snap_to_tier(params)` maps the raw parsed value to the nearest table tier:

| Raw range | Snaps to tier |
|---|---|
| 0.0 вЂ“ 1.9  | 1B  |
| 2.0 вЂ“ 5.4  | 3B  |
| 5.5 вЂ“ 9.9  | 7B  |
| 10.0 вЂ“ 19.9| 13B |
| 20.0 вЂ“ 49.9| 27B |
| 50.0+      | 70B |

Examples: `Llama-3.1-8B` в†’ 8 в†’ 7B tier; `Qwen2.5-32B` в†’ 32 в†’ 27B tier.

#### GPU VRAM Detection

`query_gpu_vram()` tries in order:

1. **`nvidia-smi --query-gpu=memory.total,name`** вЂ” most reliable; returns total VRAM in MB and GPU name
2. **`/sys/class/drm/card{0..3}/device/mem_info_vram_total`** вЂ” AMD/Intel on Linux (bytes)
3. `(None, None)` вЂ” no GPU detected; VRAM check skipped in report

#### Training Time Estimate

```
steps_per_epoch = total_samples / batch_size
total_steps     = steps_per_epoch Г— epochs
steps_per_sec   = 0.8 / (param_billions / 7.0)   (T4 baseline)
total_secs      = total_steps / steps_per_sec
```

Output format: `"44s"`, `"18m"`, `"2h 18m"`. Always labeled `(based on T4 baseline)`.

#### `vram_suggestions()`

Returns up to 4 suggestions when `fits == false`:

- Enable QLoRA вЂ” when `!config.qlora`
- Reduce batch size вЂ” when `config.batch_size > 1`
- Try a smaller model вЂ” when `base_gb > 15.0`
- Use gradient checkpointing вЂ” always included

