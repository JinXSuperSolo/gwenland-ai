### `GwenMode` API

```rust
pub struct GwenMode {
    pub dry_run: bool,
    pub non_interactive: bool,
    pub json: bool,
    pub yes: bool,
}

impl GwenMode {
    // Auto-enables non_interactive when stdout is not a terminal.
    pub fn new(dry_run: bool, non_interactive: bool, json: bool, yes: bool) -> Self;

    // True when TUI / interactive path should be used.
    pub fn is_tui(&self) -> bool { !self.non_interactive && !self.json }

    // True when output must be NDJSON (newline-delimited JSON).
    pub fn is_ndjson(&self) -> bool { self.non_interactive && self.json }
}
```

### Non-Interactive Chat (`run_chat_pipe`)

When `non_interactive=true`, `gwen chat` reads one line from stdin as the prompt, streams the response from the SSE proxy, and writes output to stdout:

**Plain text mode** (`--non-interactive`, no `--json`):
```
Hello world
```

**NDJSON mode** (`--non-interactive --json`):
```
{"type":"token","content":"Hello"}
{"type":"token","content":" world"}
{"type":"done","total_tokens":42}
```

Rules:
- One JSON object per line. Never break mid-line.
- `"type":"token"` ŌĆö emitted for every non-empty SSE chunk.
- `"type":"done"` ŌĆö emitted once when the stream ends; `total_tokens` is an approximation (`content.len()/4+1` per chunk).
- No spinners, no ANSI codes, no decorative borders in either mode.

### Non-Interactive Output Rules (enforced across all commands)

| Prohibited when `non_interactive=true` | Reason |
|---|---|
| Spinners / progress bars | No TTY to animate |
| ANSI escape codes / colors | May corrupt pipe consumers |
| `[Y/n]` interactive prompts | Would block indefinitely |
| Decorative `eprintln!` banners | Pollutes stderr in scripts |

### Pipe-Safe Auto-Detection

```rust
// Inside GwenMode::new()
use std::io::IsTerminal;
let piped = !std::io::stdout().is_terminal();
Self { non_interactive: non_interactive || piped, .. }
```

Users do not need to add `-n` when piping: `gwen chat -m gwen | jq .` automatically enables non-interactive mode.
