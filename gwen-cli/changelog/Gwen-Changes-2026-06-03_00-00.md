## Architecture Notes

| Item | Detail |
|---|---|
| Navigation | Single `useState<Screen>` in `App.tsx` — no react-router, no URL hash. Tauri's WebView has no address bar so URL-based routing adds complexity with no user benefit. |
| Lazy loading | All four screens are `React.lazy()` behind a `<Suspense>`. The initial JS chunk is minimal; each screen loads on first navigation only. |
| No `<form>` tags | All submit actions use `onClick` handlers. Tauri's WebView suppresses default form submission behaviour but it's cleaner to avoid the element entirely. |
| SSE streaming | `EventSource` at `127.0.0.1:1136/stream` bypasses Tauri IPC entirely — no serialisation round-trip per token. The Tauri bridge is used only for one-shot `invoke()` calls. DOM updates flow into a single `setMessages` call per chunk (not a re-render of the full list). |
| Glass performance | `backdrop-filter` is applied only to floating panels (Chat glass card). The sidebar uses solid `var(--bg-surface)` — no glass — per performance rule #10. Maximum 2 backdrop-filter layers in any z-stack; blur capped at 16px. |
| `will-change` scope | Only set on the `.bg-ambient` animated gradient (`will-change: background-position`). Not set globally or on static elements. |
| Font files | Geist and Geist Mono woff2 files must be manually placed in `public/fonts/`. They are not shipped in the repo; download from the Geist GitHub releases page. |
| Mock data coherence | Stub sessions in `dashboard.rs` reference the same Mistral-7B model shown as active in `get_stats()` and `get_config()`. `get_history()` returns a Tauri 1 vs 2 exchange that matches the real domain. |
| `gwen-core` dep | `default-features = false` intentionally omitted here. GUI only needs the public API surface, not GPU/candle backends that would inflate binary size significantly. |
