# GwenCLI Change Log — 2026-06-01 18:00

## Overview

Five candle-native ML infrastructure modules added to `gwen-core` this session, laying the complete Rust-native LoRA training stack: dependency declarations, config structs, the LoRA adapter layer, a JSONL dataset loader, a pure-function VRAM/time estimator, and the core training loop with gradient accumulation. No existing code was removed or broken; all additions are gated behind the `candle` feature flag and compile cleanly against the existing `gwen-core` crate.

## New / Modified Files

| File | Change |
|---|---|
| `packages/core/Cargo.toml` | Added `candle-core = { version = "0.9", optional = true }`, `candle-nn = { version = "0.9", optional = true }`, `safetensors = { version = "0.4", optional = true }` to `[dependencies]`. Added two new feature flags: `candle` (activates candle-core + candle-nn + safetensors) and `cuda` (activates `candle` + `candle-core/cuda`). CUDA feature gracefully omits the CUDA backend if no toolkit is found at compile time — build never fails on CPU-only machines. `tokenizers` and `hf-hub` were already present; unchanged. |
| `packages/core/src/train/config.rs` | Added `LoraConfig` struct with `Default` (`r=8`, `alpha=16.0`, `dropout=0.05`, `target_modules=["q_proj","v_proj"]`). Added `NewTrainConfig` struct with `Default` (`epochs=3`, `batch_size=1`, `grad_accum=16`, `lr=1e-4`). Added `Cli` stub and `From<Cli> for NewTrainConfig` — local mirror of `TrainArgs` fields so the impl compiles without a circular dependency; TUI crate will convert `TrainArgs → Cli` when wiring up. Existing YAML-backed `TrainConfig` untouched. |
| `packages/core/src/train/lora.rs` | **New file.** `LoraLayer { base: Linear, lora_a: Linear, lora_b: Linear, scale: f32 }`. `base` is constructed from a raw `Tensor` passed in by the caller and immediately `.detach()`-ed — permanently removed from the autograd graph. `lora_a` initialised via `vb.get_with_hints(…, Init::Randn { mean: 0.0, stdev: 1.0 })` (random normal). `lora_b` initialised via `vb.get_with_hints(…, Init::Const(0.0))` (zeros). Both LoRA weights are stored in the `VarMap` behind `vb` so the optimizer can reach them. `scale = alpha / r`. `forward(x)` computes `base(x) + lora_b(lora_a(x)) * scale`. `trainable_params()` returns `lora_a.weight().elem_count() + lora_b.weight().elem_count()` — base excluded. |
| `packages/core/src/train/dataset.rs` | **New file.** `Sample { input: String, output: String }`. `load_jsonl(path)` — BufReader line-by-line, skips blank lines silently, emits `eprintln!("[warn] …")` for read errors and malformed JSON, returns `Err` if zero samples survive. `tokenize(samples, tokenizer, max_len, device)` — formats each sample as `"<input>\n<output>"`, calls `tokenizer.encode(text, true)`, slices to `max_len`, builds a 1-D `u32 Tensor`. `avg_token_length(tensors)` — sums `elem_count()`, integer-divides, returns 0 for empty. `batch(tensors, batch_size)` — chunks by `batch_size`, last chunk may be smaller, `batch_size` clamped to 1. `DEFAULT_MAX_LEN = 1024`. 6 unit tests covering load, skip, empty error, batch splitting, avg length. |
| `packages/core/src/diagnostics/estimator.rs` | **New file.** `VramEstimate { model_mb, activations_mb, optimizer_mb, total_mb }`. `Device` enum with `tflops()` — CPU (0.1), T4 (65.0), A100 (312.0), RTX3080 (119.0), RTX4090 (330.0), Unknown (1.0). `estimate_vram(total_params, lora_params, batch_size, seq_len, d_model, num_layers)` — pure, no side effects: `model_mb = (params×2)/1_048_576`, `activations_mb = (batch×seq×d_model×layers×2)/1_048_576`, `optimizer_mb = (lora_params×8)/1_048_576`, `total_mb = (sum)×1.2`. `estimate_time(total_tokens, epochs, device_tflops)` — pure: `flops = tokens×epochs×6`, `effective = tflops×0.6×1e12`, returns `std::time::Duration`. Distinct from the existing `train::vram::VramEstimate` (GB-based, GPU-polled) — no name collision. 7 unit tests. |
| `packages/core/src/train/training_loop.rs` | **New file.** `TrainingLoop { config: NewTrainConfig, model: LoraLayer, batches: Vec<Vec<Tensor>>, varmap: VarMap, adamw: AdamW }`. `new(config, model, batches, varmap)` — seeds AdamW from `varmap.all_vars()` (only lora_a + lora_b; base never in VarMap). `run()` — for each epoch/batch: clone batch out of `self.batches` by index (borrow-checker split), pad+shift tokens into `(input, target)` pair via `prepare_batch`, forward through `LoraLayer`, cross-entropy loss, extract scalar before `loss_full` is moved by division, scale by `1/grad_accum`, `.backward()`, collect `GradStore`. Every `grad_accum` batches (or at final batch): call `step_accumulated` which temporarily divides lr by `n`, calls `AdamW::step` once per store, restores lr — numerically equivalent to averaging gradients. Every 500 optimiser steps: `varmap.save(checkpoint_NNNNNN.safetensors)`. Emits `{"event":"step","epoch":…,"step":…,"loss":…,"elapsed_secs":…}` JSON to stdout every batch; emits `{"event":"done",…}` on completion. Returns `TrainResult { final_loss, total_steps, elapsed }`. |
| `packages/core/src/train/mod.rs` | Added `pub mod dataset;`, `pub mod lora;`, `pub mod training_loop;`. Existing modules untouched. |
| `packages/core/src/diagnostics/mod.rs` | Added `pub mod estimator;`. Existing modules untouched. |

## Architecture Notes

| Item | Detail |
|---|---|
| CUDA feature graceful disable | `cuda = ["candle", "candle-core/cuda"]` in `[features]`. candle-core's build script detects the CUDA toolkit at compile time and skips GPU kernels if absent — the feature can be declared without failing on CPU-only machines. |
| `base` weight freezing | Candle has no `requires_grad` flag. `.detach()` is the correct mechanism — it permanently cuts the tensor out of the autograd graph. A detached tensor is never inserted into a `VarMap`, so the optimizer never receives it. |
| Borrow-checker split in `run()` | `for batch in &self.batches` would hold an immutable borrow on `self` across the entire body, blocking `&mut self` for `step_accumulated` and `save_checkpoint`. Fix: iterate by index, clone the `Vec<Tensor>` out immediately, drop the borrow before the mutable calls. |
| Gradient accumulation strategy | Candle produces a fresh `GradStore` per `.backward()` — there is no persistent gradient buffer. To simulate accumulation: collect `n` GradStores, temporarily set `lr /= n`, call `AdamW::step` once per store, restore `lr`. Equivalent to averaging because AdamW scales the update by `lr`. |
| `loss_full` move | Candle's `Div` impl for `Tensor` takes ownership of the LHS. `scalar_f32(&loss_full)` must be called before `loss_full / grad_accum as f64`, not after. |
| `NewTrainConfig` vs `TrainConfig` | `TrainConfig` is the YAML-backed Python-subprocess config. `NewTrainConfig` is the candle-native config. They coexist in `config.rs` without overlap; the naming distinction prevents silent misuse. |
| `VramEstimate` naming | `diagnostics::estimator::VramEstimate` is MB-based and purely computed. `train::vram::VramEstimate` is GB-based and GPU-polled (nvidia-smi). Different modules, different types, no collision. |
| Checkpoint format | `output_path/checkpoint_NNNNNN.safetensors` (zero-padded 6 digits). Saves only LoRA weights (the VarMap contents). Base model is reloaded from the original safetensors file. |
| `DEFAULT_MAX_LEN` | Exported constant (`= 1024`) matching the existing `TrainConfig.max_seq_len` default, so callers can reference one source of truth. |
