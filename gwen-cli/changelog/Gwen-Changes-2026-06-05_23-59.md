# Cycle 6 — Native Inference: Full Candle Migration
**Date:** 2026-06-05  
**Branch:** `jinxsuperdev/cycle-6-native-inference-candle`

---

## Summary

Cycle 6 ships native Rust inference — `gwen run` — replacing Ollama completely.
Candle is now always compiled (no feature flag). No external runtime is required.
Zero Python. Zero subprocess. Your machine. Your models. Your rules.

## What Was Added

### `engine/inference/` — new module

| File | Purpose |
|---|---|
| `mod.rs` | Module root |
| `loader.rs` | GGUF file loading, device auto-detect (CUDA → CPU), HF tokenizer fetch |
| `model_dispatch.rs` | Read `general.architecture` GGUF key → `ModelKind` enum dispatch |
| `sampler.rs` | Temperature, top-p nucleus sampling, repetition penalty |
| `runner.rs` | Token-by-token generation loop, MemoryGuard integration, dry-run table |

### `engine/memory_guard.rs` — GWEN-208

RAM watchdog checked every 16 tokens during inference. Halts generation when
system RAM ≥ `--auto-stop` threshold. Minimum threshold enforced at 10%.

### `commands/run.rs` — `gwen run <model>`

New CLI command supporting:
- `--prompt <text>` — single-shot generation
- `--interactive` — REPL chat loop (stdin/stdout)
- `--dry-run` — pre-flight table without executing
- `--temperature`, `--top-p`, `--repeat-penalty`, `--max-tokens`
- `--auto-stop <pct>` — RAM safety threshold (default: 90%)

## Architecture: Before → After

```
Cycle 5 (3 paths):
  gwen run → detect runtime → Ollama (HTTP 11434) / LlamaCpp / LmStudio / mistralrs (subprocess)

Cycle 6 (1 path):
  gwen run → inference::runner → candle-transformers → tokens to stdout
  gwen serve → platform::proxy (Actix :1136) → inference::runner → SSE tokens
```

The SSE format on `/gwenland/chat` is identical — GUI and `gwen chat` require zero changes.

## Notes

- Metal feature is excluded from `candle-core` at compile time to prevent
  `objc2` (macOS SDK) from being pulled in on Windows/Linux. Metal acceleration
  is available on macOS via `candle-core`'s runtime detection (`Device::new_metal`).
- Binary size: candle-transformers adds ~3–5 MB to the release binary. GGUF weights
  are loaded from disk at runtime, not embedded — binary stays well under 15 MB.
- The `Provider::Ollama` enum variant in `tui/main.rs` is a CLI display value
  retained for backward-compatibility with existing config files; it has no
  runtime effect in Cycle 6.
