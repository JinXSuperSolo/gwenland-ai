## Dry-Run Behaviour Per Command

### `gwen train --dry-run`

Validation steps (read-only):
1. Parse YAML config (`config.validate()`)
2. Dataset validation — reuses JIN-169 validator
3. Model in local registry or HF Hub (HEAD request)
4. Output dir free space (≥10 GB check)
5. VRAM estimate (`estimate_vram()`) and training time estimate

**Table output** (default):
```
┌─────────────────────────────────────────┐
│  gwen train --dry-run                   │
│                                         │
│  ✦ config:          (loaded)         ✓  │
│  ✦ Dataset:         dataset.jsonl    ✓  │
│  ✦ Base model:      mistralai/Mis…   ✓  │
│  ✦ Output dir:      ./output         ✓  │
│  ✦ VRAM Breakdown                       │

│                                         │
```

## Exit Codes

| Code | Constant | Meaning |
|---|---|---|
| 0 | `EXIT_READY` | All checks passed — ready to run |
| 1 | `EXIT_VALIDATION_FAILED` | One or more checks failed |
| 2 | `EXIT_INSUFFICIENT_RESOURCES` | VRAM or disk below required threshold |

---

## Safety Invariants (enforced)

| Rule | How enforced |
|---|---|
| No file writes | All dry-run paths use `find_*` / `check_disk_space` (read-only) |
| No process spawns | `dry_run_serve` only calls `find_mistralrs()` (PATH lookup, no exec) |
| No network calls | `dry_run_hub_pull` uses `Cache::from_env()` and keyring (local only). `dry_run_serve` uses `TcpListener::bind` (local socket, not outbound) |
| Port probe is a bind test | `TcpListener::bind(("127.0.0.1", port)).is_ok()` — immediately dropped after check |
